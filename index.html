Architecture

9-Step User Lifecycle Pipeline

Every user creation runs 9 sequential steps — each validated, logged, and dry-run capable. From CSV to fully configured account in under 30 seconds.

01
Parse CSV
Validate header, skip blanks & comments, trim whitespace
02
Validate Fields
Username regex, shell in /etc/shells, date format check
03
Create Groups
groupadd for any missing groups before user creation
04
useradd
Create account with home dir, shell, GECOS, secondary groups
05
Set Password
openssl rand 12-char password, chpasswd, force change on login
06
Password Aging
chage: min/max/warn/inactive days from config.conf
07
Set Expiry
chage -E expiry_date if provided in CSV column
08
Deploy SSH Key
Validate & append public key to authorized_keys, chmod 700/600
09
Log & Report
Append to /var/log/usermgmt.log, credential summary output

Module Reference

8 production modules — every lifecycle event

Each module runs standalone or is sourced by the interactive menu. All support --dry-run and write to the central audit log.

create_users.sh
modules/create_users.sh

Bulk account creation from CSV. Creates groups, runs useradd, sets a secure random password, applies aging policy, sets expiry, and deploys SSH key — all in one pass.

root dry-run logging CSV-driven SSH keys
disable_users.sh
modules/disable_users.sh

Locks accounts via usermod -L, sets expiry to epoch 1, kills active sessions with pkill -SIGTERM/-SIGKILL, and stamps a GECOS audit trail. Supports bulk CSV input.

root dry-run logging email alert
delete_users.sh
modules/delete_users.sh

Permanent account removal with safety guards: refuses system UIDs, refuses to delete the current sudo user, backs home directory to /var/backup/usermgmt/homes/ before deletion.

root dry-run logging
audit_report.sh
modules/audit_report.sh

Generates a dark-themed HTML report with 7 sections: all users, sudo users, expiring accounts, inactive accounts, never-logged-in, failed SSH logins, and sudo command history.

logging HTML output email delivery
ssh_key_manager.sh
modules/ssh_key_manager.sh

Full SSH key lifecycle: deploy (validate → deduplicate → append), list with fingerprints, revoke all, rotate (revoke + deploy new). System-wide audit across all regular users.

root dry-run logging SSH keys
expire_accounts.sh
modules/expire_accounts.sh

Scans all regular users for expired or soon-to-expire accounts. Auto-disables expired accounts via usermod -L and kills sessions. Supports manual set-expiry and extend actions.

root dry-run logging email alert
password_policy.sh
modules/password_policy.sh

Writes /etc/security/pwquality.conf (minlen, dcredit, ucredit, dictcheck) and applies chage aging (min/max/warn/inactive days) to all or specific users.

root dry-run logging PAM
set_permissions.sh
modules/set_permissions.sh

Group membership management (usermod -aG, gpasswd -d), POSIX ACL management (setfacl/getfacl), and SGID shared directory setup with default ACL inheritance.

root dry-run logging

Security & Reliability

Built for production from day one

Every design decision prioritises safety, auditability, and reversibility. No surprises in production.

🧪

Dry-Run Mode

Every module supports --dry-run. All commands are logged as "Would run: …" without executing. Test entire CSV manifests before touching production.

📋

Full Audit Trail

Every action is timestamped and appended to /var/log/usermgmt.log with level, module, and operator (via $SUDO_USER). Weekly HTML reports keep history visible.

🔐

Secure Passwords

Passwords generated via openssl rand guarantee entropy. Minimum 12 chars with diversity padding. Credentials printed once then never stored.

🛡

SSH Key Validation

Every key is validated with ssh-keygen -l before deployment. Fingerprints are logged. Duplicate keys skipped automatically. Revoke with one command.

📁

Home Backup on Delete

Before any account deletion, the home directory is archived to /var/backup/usermgmt/homes/username_TIMESTAMP.tar.gz. Recoverable for 90 days.

🚫

Safety Guards

Refuses to delete system accounts (UID < 1000), refuses to delete the current $SUDO_USER, and requires typing the username to confirm destructive actions.

📧

Email Alerts

Configurable email notifications for account disabling and expiry events. Integrates with system mail command — no external dependencies required.

🔄

Log Rotation

Installer configures logrotate: weekly rotation, 90-day retention, compressed archives, dateext naming. Systemd timer or cron for scheduled reports.
