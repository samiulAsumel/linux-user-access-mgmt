04 / RUN
sudo user_manager

Architecture

9-Step User Lifecycle Pipeline

Every user creation runs 9 sequential steps — each validated, logged, and dry-run capable. From CSV to fully configured account in under 30 seconds.

01
Parse CSV
Validate header, skip blanks & comments, trim whitespace
02
Validate Fields
Username regex, shell in /etc/shells, date format check
03
Create Groups
groupadd for any missing groups before user creation
04
useradd
Create account with home dir, shell, GECOS, secondary groups
05
Set Password
openssl rand 12-char password, chpasswd, force change on login
06
Password Aging
chage: min/max/warn/inactive days from config.conf
07
Set Expiry
chage -E expiry_date if provided in CSV column
08
Deploy SSH Key
Validate & append public key to authorized_keys, chmod 700/600
09
Log & Report
Append to /var/log/usermgmt.log, credential summary output

Audit & Compliance

Weekly HTML audit report — always ready

Auto-generated every Monday morning. Seven sections covering every access control concern an auditor would ask about.

§ 1
All Users
Username, full name, UID, shell, groups, last login, lock status
§ 2
Sudo Users
All wheel/sudo group members with their privilege level
§ 3
Expiring Soon
Accounts expiring within 30 days — colour-coded urgency
§ 4
Inactive Accounts
No login in 90+ days — ideal candidates for disabling
§ 5
Never Logged In
Provisioned accounts with zero login activity
§ 6
Failed Logins
Last 7 days SSH failures per user/IP via journalctl
§ 7
sudo Usage
Last 50 sudo commands from journal or /var/log/secure
